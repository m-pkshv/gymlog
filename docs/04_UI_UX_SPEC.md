# 04_UI_UX_SPEC — Спецификация UI/UX

## 1. Назначение документа
Определяет карту экранов, навигацию, состояния экранов (загрузка/пусто/ошибка), подтверждения, модальные окна, клавиатуру, жесты и их альтернативы, адаптацию, доступность, темы и правила локализации. Бизнес-правила — `03_TECHNICAL_SPEC.md`; поля и валидация — `06_DATA_MODEL.md`.

## 2. Область ответственности
Поведение и структура интерфейса. Не входит: пиксельные макеты (создаются в коде на базе Material 3 с токенами раздела 9), формулы и хранение.

## 3. Принципы
1. Контекст спортзала (`01`, раздел 4): крупные касания (мин. 48×48 dp), важные действия — в нижней половине экрана, минимум обязательного ввода.
2. Ни одно действие не выполняется **только** жестом (ТЗ): у каждого жеста есть кнопочная альтернатива (раздел 8).
3. Автосохранение: интерфейс не содержит кнопок «Сохранить» в редакторах подходов/тренировки; изменения фиксируются сразу (`03`, раздел 5).
4. Деструктивные действия — подтверждение или Undo (`06`, раздел 10).
5. Material 3, `ColorScheme.fromSeed`, системные компоненты; кастомные виджеты — только при отсутствии стандартного.

## 4. Карта экранов и навигация
Нижняя навигация (5 вкладок): **Сегодня · История · Упражнения · Статистика · Ещё**.
```
Сегодня (S-01)
История (S-02) ─ Редактор тренировки (S-03) ─ Активная тренировка (S-04) ─ Итог (S-05)
Упражнения (S-06) ─ Карточка упражнения (S-07) ─ Форма упражнения (S-08, модаль)
Статистика (S-09) ─ Прогресс по упражнению (S-10)
Ещё (S-11): Шаблоны (S-12) ─ Редактор шаблона (S-13)
            Измерения тела (S-14) ─ Форма измерения (S-15, модаль)
            Импорт/экспорт (S-16)
            Настройки (S-17)
```
- Маршруты go_router: `/today`, `/history`, `/history/workout/:id`, `/active`, `/active/summary`, `/exercises`, `/exercises/:id`, `/stats`, `/stats/exercise/:id`, `/more`, `/more/templates`, `/more/templates/:id`, `/more/measurements`, `/more/export`, `/more/settings`.
- Активная тренировка (S-04) — полноэкранный маршрут поверх вкладок; выход — «Завершить», «Отменить» или системный «назад» (сворачивает в баннер «Тренировка идёт» на всех вкладках, не прерывая режим).
- Кнопка «назад» Android везде ведёт на предыдущий экран; на корне вкладки — стандартное поведение выхода.

## 5. Спецификации экранов
Формат: назначение → ключевые элементы → действия → особые состояния. Общие состояния — раздел 6.

**S-01 Сегодня.** Ближайшая тренировка (сегодня/будущая ближайшая): карточка со статусом, упражнениями, кнопки «Начать» / «Открыть»; при активной — карточка «Продолжить». Ниже — быстрые действия: «Новая тренировка», «Из шаблона», «Скопировать прошлую». Пустое состояние: приветствие + те же действия.

**S-02 История/Календарь.** Переключатель «Список / Календарь» (месяц с точками-статусами). Фильтры (bottom sheet): диапазон дат, текст по названию, статусы (мультивыбор), теги (мультивыбор; скрытые теги не показываются в фильтре при `showTags = 0`). Список: пагинация по 50, карточка = дата, имя, статус-чип, теги, число упражнений, длительность. Действия по карточке: открыть; меню «⋮»: копировать, перенести (date picker), удалить (Undo). FAB «Новая тренировка».

**S-03 Редактор тренировки.** Шапка: дата (нажатие — перенос), имя, статус-чип с меню переходов (только разрешённые — `06`, 6.4.1), теги (чипы + «+»), комментарий. Список упражнений: карточки с превью подходов; reorder — ручка-иконка (drag) и меню «⋮ → Вверх/Вниз». В карточке упражнения: подходы таблицей (№, план, факт, ✓), кнопки «+ Подход», «Копировать предыдущий подход», меню подхода (удалить — Undo, комментарий); «Копировать показатели прошлого выполнения»; «Прошлые результаты» (bottom sheet с последними 5 выполнениями); «Сведения» (→ S-07); решение о прогрессии (сегмент: —/↑/=/↓) с подсказкой «N без роста»; комментарий упражнения; удалить упражнение (Undo). Кнопка «+ Упражнение» → выбор из каталога (поиск/фильтры) с кнопкой «Создать новое» (S-08 модально; после создания сразу добавляется). Кнопки «Начать тренировку» (если не `completed`) / «Завершить».

**S-04 Активная тренировка.** Верх: таймер тренировки, пауза/продолжить, «Завершить». Центр: текущее упражнение с подходами; отметка «✓» на подходе — крупная (56 dp). Быстрое редактирование факта: тап по значению → числовая клавиатура-степпер (±шаг, ввод). Таймер отдыха: появляется после отметки подхода (если автозапуск), полоса с остатком, ±15 с, пропустить. Список остальных упражнений — свёрнутые секции. Добавление упражнения/подхода доступно. Поведение при сворачивании/восстановлении — `03`, раздел 7.

**S-05 Итог тренировки.** Длительность, число упражнений/подходов, тоннаж, новые рекорды (если есть), поле комментария, решения прогрессии по упражнениям (можно проставить здесь), кнопка «Готово» → История.

**S-06 Упражнения.** Поиск (по имени, все локали имени), фильтры: тип, мышечная группа, оборудование, «показывать архивные» (выкл. по умолч.), «только пользовательские». Список: имя, тип-иконка, группа мышц, бейдж «архив». FAB «Создать». Тап → S-07.

**S-07 Карточка упражнения.** Изображение (или плейсхолдер), имя, тип, группы мышц, оборудование, описание, кнопка YouTube (внешний браузер, диалог «Откроется браузер» при первом разе). Вкладки: «О упражнении» / «История» (последние выполнения) / «Рекорды». Действия: редактировать (пользовательское; встроенное — только архивировать), архивировать/разархивировать, удалить (только неиспользованное пользовательское, `06`, раздел 10).

**S-08 Форма упражнения (модаль).** Поля: имя (обяз.), тип (неизменяем при наличии истории — поле блокировано с пояснением), группа мышц, доп. группы (мультивыбор), оборудование, метрика усилия (для силового), описание, YouTube. Кнопка «Создать»/«Сохранить» активна при валидном имени.

**S-09 Статистика.** Секции-карточки: Вес тела, % жира, Замеры (выбор типа), Тренировки (число/частота/тоннаж), «Прогресс по упражнению» (поиск упражнения → S-10). Каждый график: переключатель периода (Нед/Мес/3М/Год/Всё/Свой — range picker), значения в единицах настройки.

**S-10 Прогресс по упражнению.** Для strength/reps: графики макс. веса, расчётного 1ПМ (пометка «расчётный»), тоннажа; таблица «повторения при весе». Для cardio: дистанция, темп, длительность. Рекорды списком (значение, дата, ссылка на тренировку).

**S-12/S-13 Шаблоны.** Список (имя, число упражнений, меню: создать тренировку, дублировать, архивировать, удалить). Редактор — как S-03, но без статусов/дат/фактов; только структура и плановые значения. «Создать тренировку из шаблона» → выбор даты → открывается S-03.

**S-14/S-15 Измерения.** Вкладки по типам (Вес, % жира, Замеры, Свои). Список записей по датам; график сверху. «+» → форма: тип, дата (def сегодня), значение (единицы настройки), комментарий. Дубликат за день — диалог «Заменить существующее значение?». Управление пользовательскими типами — здесь же («Добавить замер…»).

**S-16 Импорт/экспорт.** Кнопка «Экспортировать данные (CSV)» → прогресс → системный share. Журнал операций (из `ImportExportOperation`): дата, статус, счётчики. Раздел «Импорт» — заглушка «Появится в следующих версиях». Ссылка «Описание формата CSV» (встроенная справка из `03`, раздел 10).

**S-17 Настройки.** Система единиц (метрическая/имперская), тема (системная/светлая/тёмная), язык (системный/RU/EN), «Показывать теги» (переключатель; выключение скрывает чипы и фильтр тегов, данные не меняются), таймер отдыха по умолчанию + автозапуск, «Уведомления» (статус + переход в системные настройки), «О приложении» (версия, версия формата экспорта).

## 6. Общие состояния экранов
| Состояние | Паттерн |
|---|---|
| Загрузка | Первичная — центрированный `CircularProgressIndicator` на весь экран (UX-3, Этап 9: упрощение исходного требования skeleton-плейсхолдеров, подтверждено владельцем); операции — линейный прогресс/disabled кнопка со спиннером |
| Пустое | Иллюстрация-иконка + 1 строка объяснения + основное действие (каждый список обязан иметь дизайн пустого состояния); экраны-пикеры (выбор источника для копирования и т.п.) могут ограничиться текстом без иконки/действия — там нет отдельного «основного действия» кроме возврата назад |
| Ошибка | Хранилище: экранная заглушка «Не удалось загрузить» + «Повторить» (перечитывает данные заново, не просто обновляет экран); операционные ошибки — snackbar с текстом из `AppError` (локализованным); ошибка миграции — блокирующий экран (`06`, раздел 11) |
| Подтверждения | Деструктивные без Undo — AlertDialog (заголовок-вопрос, «Отмена» слева, действие справа, деструктивное — красным); с Undo — snackbar 5 с «Удалено · ОТМЕНИТЬ» |
| Модальные окна | Формы создания — полноэкранные модали (safe area); выборы/фильтры — bottom sheet с ручкой-индикатором |

## 7. Клавиатура и ввод
- Числовые поля — `TextInputType.numberWithOptions(decimal:)`; принимаются `.` и `,` (`03`, раздел 6).
- Открытая клавиатура не перекрывает активное поле: `Scaffold.resizeToAvoidBottomInset = true` + автопрокрутка к полю (`ensureVisible`).
- «Done/Next» на клавиатуре: Next между полями формы, Done подтверждает и снимает фокус (триггер автосохранения).
- В таблице подходов ввод — через степпер-панель (S-04) либо инлайн-поле (S-03); tap-мишень значения ≥ 48 dp.

## 8. Жесты и обязательные альтернативы
| Жест | Альтернатива (обязательна) |
|---|---|
| Drag-and-drop порядка упражнений | Меню «⋮» → «Переместить вверх/вниз» |
| Swipe для удаления записи в списках | Меню «⋮» → «Удалить» |
| Swipe между вкладками карточки | Тапы по вкладкам |
| Long-press для мультидействий | Не используется в MVP |
Все свайпы дублируются; обучение жестам не требуется.

## 9. Темы и стиль
- Material 3. Seed-цвет: `#4C7BD9`. Светлая и тёмная темы — из одного seed (`ColorScheme.fromSeed(brightness:)`); режим — из настройки (def системный).
- Семантические цвета статусов (одинаковы в обеих темах по роли, тон адаптируется): draft — серый, planned — синий, inProgress — оранжевый, completed — зелёный, skipped — жёлто-серый, cancelled — красный-приглушённый.
- Палитра тегов (8 цветов на выбор): `#4C7BD9 #2E9E6B #D9774C #B34CD9 #D9B84C #4CC3D9 #D94C6B #7B8794`.
- Типографика — системная (Roboto/SF); масштаб текста пользователя уважается до 1.6× без поломки макетов (проверка в чек-листах этапов).
- Запрещено кодировать цвета в виджетах напрямую — только через `Theme`/`ColorScheme` и токены выше.

## 10. Адаптация под размеры экранов
- Базовый диапазон: ширина 320–480 dp (телефоны), портрет. Макеты — резиновые: `Expanded/Flexible`, без фиксированных ширин контента.
- Малые экраны (320 dp): таблица подходов сокращает подписи до иконок; проверка в ручных чек-листах.
- Планшеты: то же телефонное расположение с ограничением ширины контента 600 dp по центру. Safe area учитывается на всех экранах (вырезы, home-индикатор).

## 11. Доступность (accessibility)
- Все интерактивные элементы: мин. 48×48 dp, `Semantics`-подписи (иконки без текста обязаны иметь label).
- Поддержка TalkBack/VoiceOver: порядок фокуса соответствует визуальному; отметка подхода озвучивается («Подход 2, выполнен»).
- Контраст текст/фон ≥ 4.5:1 в обеих темах (проверять токены).
- Динамический размер шрифта — раздел 9. Состояния передаются не только цветом (статусы имеют текст/иконку).

## 12. Локализация
- ARB (`app_ru.arb`, `app_en.arb`); ключи — англ. camelCase; плюрализация через ICU (`{count, plural, ...}`) — обязательна для «N тренировок/подходов/дней».
- Русские строки длиннее английских в ~1.3×: макеты проверяются на RU; переполнение — `TextOverflow.ellipsis` + полный текст в tooltip/деталях; переносы не ломают кнопки (мин. высота фикс.).
- Даты/числа отображения — через `intl` по локали приложения; **хранение и CSV — независимо от локали** (`03`, раздел 10).
- Встроенные упражнения/группы мышц/оборудование — локализуются (ключи `06`, раздел 5). Пользовательский контент не переводится.
